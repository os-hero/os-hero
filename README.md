# OS Hero

OS Hero is a macOS Menu Bar and Windows System Tray pixel character app inspired
by RunCat. The tray icon is a small 2D pixel human character whose idle animation
gets faster as CPU usage rises.

## Repositories

- App source: https://github.com/os-hero/os-hero
- Public website and update feed: https://github.com/os-hero/os-hero.github.io

Build and release commands are run from this app source repository. The deploy
script copies generated release artifacts into the website/update-feed
repository.

## Tech Stack Decision

The MVP uses Electron.

- One JavaScript codebase supports macOS Menu Bar and Windows System Tray.
- Electron's `Tray` API supports stable icon frame replacement with generated
  PNG frames.
- CPU usage is sampled in the main process with Node's `os.cpus()` data every
  second.
- Local data is stored as JSON files under Electron's `userData` directory.
- Login auto-start is handled with Electron's OS-backed
  `app.setLoginItemSettings`.
- Installable packages are generated with `electron-builder`.
- Packaged Electron locale resources are limited to English, Korean, and
  Simplified Chinese so GitHub Pages update artifacts stay below GitHub's
  normal file size limits.

Tauri was considered, but this workspace does not currently have the Rust
toolchain installed. Electron lets the MVP run and package from the available
Node environment while still meeting the tray/menu bar requirements.

## Requirements

- Node.js 20.17+ is recommended for packaging.
- Node.js 18 can run the app in this workspace, but newer packaging dependencies
  may print engine warnings during `npm install`.

## Run

```bash
npm install
npm start
```

On first launch, the tray/menu bar icon appears immediately with the default
character, and the customization window opens automatically.

## Build

```bash
npm run pack
npm run dist
```

Platform-specific builds:

```bash
npm run dist:mac
npm run dist:win
```

Build output is written to `release/`.
`npm run dist:win` builds Windows x64 installers by default for distribution.

The packaged app icon is generated from `build/icon.png`.

- macOS package icon: `build/icon.icns`
- Windows package icon: `build/icon.ico`
- Runtime window icon: `public/assets/app-icon.png`

## Updates

OS Hero uses `electron-updater` with a generic HTTP update feed.

Runtime behavior:

- On app launch, OS Hero checks the update feed once.
- If a newer version is available at launch, OS Hero downloads it in the
  background.
- The app does not poll for updates while it is already running.
- A downloaded startup update is applied on app quit, or immediately when the
  user opens Settings and chooses to apply it.
- In Settings, `새 버전 확인` manually checks for a newer version.
- If a newer version exists, the Settings update area shows an update action.
- `업데이트 진행` downloads the update and restarts the app into the new
  version.

The production update feed is hosted on GitHub Pages:

```json
{
  "provider": "generic",
  "url": "https://os-hero.github.io/updates/"
}
```

`electron-builder` writes this feed URL into packaged builds through
`app-update.yml`. A client can only auto-update remotely after it has installed
at least one build that contains this production feed URL.

For local testing, the feed can be overridden at runtime:

```bash
OS_HERO_UPDATE_URL="https://your-domain.example/os-hero/" npm start
```

For packaged builds, an external file can also be placed next to the packaged
resources:

```json
{
  "url": "https://your-domain.example/os-hero/"
}
```

File name:

```text
update-feed.json
```

Generic feed hosting must expose the files generated by `electron-builder`, for
example:

macOS:

```text
latest-mac.yml
OS Hero-0.1.13-arm64-mac.zip
OS Hero-0.1.13-arm64.dmg
```

Windows:

```text
latest.yml
OS Hero Setup 0.1.13.exe
```

Release deployment flow:

```bash
npm version <next-version> --no-git-tag-version
npm run dist:mac
npm run dist:win
npm run deploy:updates
```

`npm run deploy:updates` clones or updates
`https://github.com/os-hero/os-hero.github.io.git`, copies the current
version's update artifacts into `updates/`, refreshes `latest*.yml`, commits
the change, and pushes the GitHub Pages repository. Versioned installers are
preserved so the website archive can keep older downloads available. Set
`OS_HERO_PAGES_REPO` if you prefer the SSH remote. The public feed URL is:

```text
https://os-hero.github.io/updates/
```

For a real macOS release, sign and notarize the app. For Windows, sign the NSIS
installer. Auto-update is much more reliable when releases are signed.

## macOS Install and Run

1. Build on macOS with `npm run dist:mac`.
2. Open the generated `.dmg` or `.zip` from `release/`.
3. Move `OS Hero.app` to Applications and run it.
4. The app lives in the Menu Bar. Use the Menu Bar icon to open Hero,
   inventory, quests, settings, app info, or quit.

For production distribution, add Apple Developer signing and notarization to the
`electron-builder` configuration.

## Windows Install and Run

1. Build on Windows with `npm run dist:win`.
2. Run the generated NSIS installer or portable executable from `release/`.
3. The app lives in the System Tray. Use the tray icon to open Hero,
   inventory, quests, settings, app info, or quit.

Windows packages are best produced on Windows to avoid cross-build toolchain
issues.

## Local Storage

Electron's `userData` directory is used.

macOS:

```text
~/Library/Application Support/OS Hero/character.json
~/Library/Application Support/OS Hero/settings.json
~/Library/Application Support/OS Hero/quests.json
```

Windows:

```text
%APPDATA%\OS Hero\character.json
%APPDATA%\OS Hero\settings.json
%APPDATA%\OS Hero\quests.json
```

Stored character shape:

```json
{
  "hasCharacter": true,
  "gender": "male",
  "bodyColor": "#F1C27D",
  "eyeType": "default",
  "equipped": {
    "head": null,
    "clothes": "default_clothes",
    "tool": null
  },
  "version": "0.1.13"
}
```

Stored settings shape:

```json
{
  "launchAtLogin": false,
  "language": "en",
  "version": "0.1.13"
}
```

Stored quests shape:

```json
{
  "quests": [
    {
      "id": "uuid",
      "type": "adventure",
      "title": "Prepare launch notes",
      "status": "todo",
      "body": "Write down the next mission for the hero.",
      "url": "",
      "remindAt": null,
      "createdAt": "2026-05-30T00:00:00.000Z",
      "updatedAt": "2026-05-30T00:00:00.000Z",
      "notifiedAt": null,
      "version": "0.1.13"
    }
  ]
}
```

When OS Hero starts for the first time after the rename, it copies existing
`OS Boy` character and settings JSON files into the new `OS Hero` user data
directory if the new files do not already exist.

## Language

The first launch always defaults to English. The Menu Bar / System Tray menu has
a `Language` submenu with:

- English
- 한글
- 中文（简体）

Changing the language saves `settings.language` and immediately refreshes the
tray menu plus any open Hero, inventory, quests, settings, or about windows.
Existing settings without a language value are normalized to English.

## Auto Launch

The Settings window toggles login auto-start.

- macOS: Electron delegates to the platform login item mechanism through
  `app.setLoginItemSettings({ openAtLogin })`.
- Windows: Electron writes the appropriate login item. In development mode it
  passes the app path as an argument to the Electron executable; packaged builds
  use the installed app executable.

## Character and Animation

The MVP generates layered PNG frames in `src/main/pixelRenderer.js` instead of
loading static GIFs. The tray/menu bar animation swaps 4 generated frames with
`Tray#setImage`.

Animation speed policy:

| CPU usage | Frame interval |
| --- | --- |
| 0-10% | 900 ms |
| 10-30% | 650 ms |
| 30-60% | 420 ms |
| 60-85% | 250 ms |
| 85-100% | 130 ms |

CPU usage is sampled every 1 second and smoothed before driving frame speed.

## Add or Replace Assets

The current MVP uses generated pixel layers so it has no external image
dependency. Future PNG assets can be added under `public/assets/` with this
layout:

```text
public/assets/
  character/
    body/
    eyes/
    clothes/
    head/
    tools/
```

Keep source frames square and small, such as 16x16 or 32x32. Render them with
nearest-neighbor scaling before passing them to `Tray#setImage`.

## Add Items

Add item records in `src/shared/catalog.js`.

Each item needs:

- `id`
- `name`
- `category`: `head`, `clothes`, or `tool`
- `slot`: `head`, `clothes`, or `tool`
- `assetPath`
- `owned`
- `style`

If the item needs a new visual layer, add drawing logic in
`src/main/pixelRenderer.js`. The inventory UI automatically lists owned items and
enforces one equipped item per slot. Equipping saves immediately and refreshes
the tray icon.

## Quests

The Menu Bar / System Tray includes a `Quests` menu between Inventory and
Settings.

- Adventure Log: memo-style quest with title, status, and body.
- Bookmark Record: title and web address. It has no status and opens the URL in
  the user's default browser from the detail view.
- Reminder: title, body, status, and reminder time. Electron schedules a local
  OS notification; clicking the notification opens the quest detail window.

Quest status values are `todo`, `in_progress`, `done`, and `on_hold`. Status can
be changed from the quest table, the read-only detail view, or the edit form.
Quest data is stored in `quests.json`.

When a reminder is saved, OS Hero checks notification support and requests
notification permission through the renderer's Web Notifications API if needed.
If permission is denied, the reminder is not saved until the user allows OS Hero
notifications in system settings. At delivery time, the app records a reminder
as notified only after Electron reports that the OS notification was shown.

Platform notes:

- macOS requires notification permission. Production builds should be signed so
  notification click events are reliable.
- Windows does not show the same app-level permission prompt, but notifications
  can still be blocked by system notification settings, Focus Assist, or policy.

## Implemented MVP Scope

- macOS Menu Bar / Windows Tray app structure through Electron.
- Tray icon is always shown while the app is running.
- Left-click and right-click open the tray/menu bar menu.
- CPU-based idle animation speed.
- Programmatic 2D pixel human character frames.
- First-run customization window.
- Body color hex input with validation.
- Eye type selection.
- Gender selection with `male` as the default stored value.
- Pixel head items for female-style hair: Long Hair, Bob Hair, Twin Tails.
- Save and cancel behavior.
- Cancel on first run saves the default character and keeps the app running.
- JSON local persistence and reload on next run.
- Inventory window with Head, Clothes, and Tools tabs.
- One equipped item per head, clothes, and tool slot.
- Quests window with Adventure Log, Bookmark Record, and Reminder types.
- Quest table with newest-first ordering, pagination, detail view, edit, delete,
  and inline status updates.
- Local OS notifications for reminder quests.
- Windows resize to the rendered content height on open while staying inside the
  OS work area.
- Settings window with Launch at login, version, developer, contact, copyright.
- English, Korean, and Simplified Chinese UI language support.
- Tray/Menu Bar `Language` submenu with the selected language checked.
- App info window.
- Quit menu item.
- Electron Builder packaging scripts.
- Update environment with `electron-updater`.
- App-launch update check without periodic polling.
- Settings update area with manual check, download progress, and restart apply.
